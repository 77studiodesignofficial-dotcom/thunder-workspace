## 二、更新策略选择

### 策略A：手动更新（推荐）
**特点**：
- 人工确认每次更新
- 更新前自动备份
- 风险评估后执行
- 失败可自动回滚

**适用**：Boss 希望控制更新时机

### 策略B：自动更新（风险较高）
**特点**：
- 自动检测并安装新版本
- 适合稳定环境
- 可能引入意外问题

**适用**：生产环境稳定后

### 策略C：延迟更新（平衡）
**特点**：
- 新版本发布 X 天后自动更新
- 等待社区验证稳定性
- 兼顾安全和稳定

**适用**：平衡方案

**建议**：先采用 **策略A（手动更新）**，系统稳定后考虑策略C。

## 三、安全更新流程（四步法）

```
┌─────────────────────────────────────────────────────────┐
│ Step 1: 检查更新                                         │
│  ./update-manager.sh check                              │
│  ├── 显示当前版本 vs 最新版本                            │
│  └── 提示是否有新版本                                   │
├─────────────────────────────────────────────────────────┤
│ Step 2: 风险评估                                         │
│  ./update-manager.sh risk                               │
│  ├── 检查活跃进程数量                                    │
│  ├── 检查网络稳定性                                      │
│  ├── 检查磁盘空间                                        │
│  └── 检查未完成任务                                      │
├─────────────────────────────────────────────────────────┤
│ Step 3: 预更新备份                                       │
│  自动执行：                                              │
│  ├── 备份 .env.skill（凭证）                             │
│  ├── 备份 MEMORY.md（记忆）                              │
│  ├── 备份 crontab（定时任务）                            │
│  ├── 备份 workspace/（所有脚本）                         │
│  └── 记录当前版本                                        │
├─────────────────────────────────────────────────────────┤
│ Step 4: 安全升级                                         │
│  ./update-manager.sh upgrade                            │
│  ├── 停止所有 OpenClaw 进程                              │
│  ├── 清理临时文件                                        │
│  ├── 执行 npm update                                     │
│  ├── 验证新版本                                          │
│  ├── 重启 Gateway                                        │
│  └── 测试基本功能                                        │
├─────────────────────────────────────────────────────────┤
│ 失败？→ 自动回滚                                          │
│  ./update-manager.sh rollback                           │
│  ├── 恢复凭证                                            │
│  ├── 恢复 MEMORY.md                                      │
│  ├── 恢复 crontab                                        │
│  ├── 恢复脚本                                            │
│  └── 降级到旧版本                                        │
└─────────────────────────────────────────────────────────┘
```

## 四、更新管理器使用指南

### 1. 检查是否有更新
```bash
cd ~/.openclaw/workspace
./update-manager.sh check
```

**输出示例**：
```
当前版本: 2026.2.26
最新版本: 2026.3.7

🔄 发现新版本！
执行 './update-manager.sh upgrade' 进行安全升级
```

### 2. 评估更新风险
```bash
./update-manager.sh risk
```

**输出示例**：
```
风险评估:
RISK_SCORE=15
RISK_LEVEL=🟢 低风险
RISK_FACTORS=✅ 系统状态良好，可以安全更新
```

### 3. 执行安全升级（四步合一）
```bash
./update-manager.sh upgrade
```

**交互过程**：
```
准备升级: 2026.2.26 → 2026.3.7

[INFO] 执行预更新备份...
[INFO] ✅ 凭证已备份
[INFO] ✅ MEMORY.md 已备份
[INFO] ✅ Crontab 已备份
[INFO] ✅ 脚本已备份

风险评估:
RISK_SCORE=15
RISK_LEVEL=🟢 低风险

确认升级? (y/N): y

[INFO] 停止 OpenClaw 进程...
[INFO] 执行 npm update...
[INFO] ✅ 升级成功
[INFO] 新版本: 2026.3.7
[INFO] 重启 OpenClaw Gateway...
[INFO] ✅ Gateway 已启动
[INFO] ✅ 基本功能正常

✅ 升级成功！
备份保留在: ~/.openclaw/backups/pre_update_20260308_034500
如需回滚，执行: ./update-manager.sh rollback
```

### 4. 回滚（如果升级后出现问题）
```bash
./update-manager.sh rollback
```

**自动执行**：
- 从最近备份恢复所有文件
- 降级到旧版本
- 重启服务

自动备份以下文件：

|-----------|------|--------|
| `~/.openclaw/.env.skill` | API Keys 和 Tokens | 🔴 关键 |

## 六、风险评估标准

| 检查项 | 风险分值 | 说明 |
|--------|---------|------|
| 活跃进程 > 2个 | +20 | 可能有任务在执行 |
| 网络不稳定 | +30 | 更新可能中断 |
| 磁盘空间 > 90% | +25 | 空间不足 |
| 未完成任务 > 0 | +15 | 可能丢失任务状态 |

| 总分 | 风险等级 | 建议 |
|------|---------|------|
| < 20 | 🟢 低风险 | 可以安全更新 |
| 20-50 | 🟡 中风险 | 建议等待或手动处理 |
| > 50 | 🔴 高风险 | 不建议更新，先解决问题 |

**更新频率**：
- 安全更新：立即安装

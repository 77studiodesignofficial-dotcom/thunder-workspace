# 双重通道备份方案

## 当前状态：单点故障风险

当前仅依赖 Telegram 单通道：
- Telegram 连接中断 → 完全失联
- 需要备用通道

## 建议方案：Telegram + 本地日志

### 通道1：Telegram（主通道）
- 实时消息推送
- 交互式命令
- 当前已配置

### 通道2：本地文件日志（备用通道）
- 即使 Telegram 中断，日志仍持续写入
- 事后可追溯
- 可配合文件同步工具（如 Dropbox/iCloud）实现离线查看

## 实施步骤

1. **所有重要消息双写**
   - Telegram 发送
   - 同时写入本地日志

2. **关键告警多渠道**
   - 掉线告警 → Telegram + 本地日志
   - 安全告警 → Telegram + 本地日志
   - 系统故障 → Telegram + 邮件（可选）

3. **状态文件标记**
   - 创建 `$HOME/.openclaw/status.json`
   - 实时更新在线/离线状态
   - Boss 可直接查看文件了解 Thunder 状态

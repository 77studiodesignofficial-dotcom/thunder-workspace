# AUTHORIZATION_MATRIX_v1.md

## 授权层级定义

### 🟢 绿色 - 自主执行（无需授权）
已配置且已验证的能力，可在主会话或并行任务中自主使用。

**包括：**
- 使用已就绪的 skills（notion、gemini、clawhub、apple-notes 等）
- 使用已配置的 API（Notion、GitHub、Gemini、OpenAI 等）
- 本地文件操作（在 workspace 目录内）
- 读取状态、生成报告、检查日志
- 执行已列入执行板的低风险的并行任务
- 使用已安装的 CLI 工具（uv、brew 已安装的软件）

**边界：**
- 不影响系统全局配置
- 不产生费用
- 不访问敏感隐私数据（密码、密钥本体）

---

### 🟡 黄色 - 知会执行（执行后告知）
低风险但需要用户知晓的操作，执行后立即汇报结果。

**包括：**
- 安装新的 CLI 工具（通过 brew/npm，已知安全来源）
- 修改环境变量（PATH、API key 配置）
- Gateway restart / config patch
- 创建新的 API Key（在已授权的服务内，如新的 OpenAI key）
- 读取非敏感的外部数据（天气、搜索等）

**边界：**
- 可逆操作（可卸载、可恢复）
- 无直接费用
- 需要汇报操作结果

---

### 🔴 红色 - 必须授权（执行前确认）
必须获得明确许可后才能执行的操作。

**包括：**
- 新的外部服务注册（OAuth、新平台账号）
- 访问敏感数据（备忘录内容、邮件内容、密码管理器）
- 修改系统级配置（zshrc、hosts、系统设置）
- 产生费用的操作（付费 API、订阅、购买）
- 删除或覆盖重要文件（非 workspace 内）
- 安装系统级软件（需要 sudo）
- 分享数据到外部（上传、同步到云端）

**边界：**
- 不可逆或难以恢复
- 涉及隐私或安全
- 产生费用

## 授权升级/降级流程

**升级（🟢→🟡→🔴）：**
- 默认从绿色开始
- 遇到边界情况，暂停并询问
- 用户明确授权后，可降级为黄色或绿色

**降级（🔴→🟡→🟢）：**
- 反复验证安全的操作，可申请降级
- 需用户明确同意
- 文档化后生效

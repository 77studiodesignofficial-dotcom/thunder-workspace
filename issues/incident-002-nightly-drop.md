## 二、可能的根本原因

### 假设A：Cron 任务阻塞导致资源耗尽

**机制**:
```
17:00 左右 Cron 触发
    ↓
任务开始执行（包括 web_search 等操作）
    ↓
web_search 因缺少 Brave API Key 失败/超时
    ↓
任务重试或挂起
    ↓
16分钟后（17:26）系统再次触发或重试
    ↓
资源累积占用，导致服务不稳定
    ↓
晚间某个时刻服务崩溃/掉线
```

### 假设B：Web Search API 调用失败连锁反应

**证据**:
- 晨报中显示 "❌ 无法获取（缺少 Brave Search API Key）"
- web_search 工具可能因缺少 API Key 而长时间超时
- 这可能导致整个 Cron 任务挂起

### 假设C：双重消息推送问题

**现象**:
- 17:10 和 17:26 两次系统消息
- 这可能意味着：
  - Telegram Bot 重复发送
  - 或者任务真的执行了两次
  - 或者系统重试机制触发
